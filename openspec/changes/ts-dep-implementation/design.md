## 📊 Поток данных

1. **Инициализация**: CLI парсит аргументы командной строки, загружает конфиг
2. **Сканирование**: Находит все пакеты в `packages/` по наличию `package.json`
3. **Анализ**: Для каждого пакета:
   - Создаёт `ts.Program` с кастомным `CompilerHost`
   - Парсит все `.ts` файлы
   - Извлекает статические импорты/экспорты из AST
4. **Resolution**: Для каждого импорта:
   - Разрешает `@scope/package` → физический путь
   - Обходит баррель-файлы (`index.ts`) до конкретных файлов
5. **Построение графа**: Создаёт узлы (файлы) и рёбра (импорты)
6. **Экспорт**: Сохраняет граф в `graph.json`
7. **Сервер**: Запускает HTTP сервер, открывает браузер

## 🔌 Модули и зависимости

### src/cli/index.ts
- **Зависимости**: `commander`, `./config/loader.ts`, `./scanner/nx.ts`, `./server/index.ts`
- **Ответственность**: Entry point, парсинг CLI аргументов, оркестрация пайплайна

### src/config/loader.ts
- **Зависимости**: `node:path`, `node:fs`
- **Ответственность**: Загрузка `ts-dep.config.ts`, валидация схемы, merge с дефолтами

### src/scanner/nx.ts
- **Зависимости**: `node:fs`, `node:path`, `./config/types.ts`
- **Ответственность**: Рекурсивный обход `packages/`, поиск `package.json`, исключение `node_modules`

### src/analyzer/parser.ts
- **Зависимости**: `typescript`, `./config/types.ts`, `./analyzer/resolver.ts`
- **Ответственность**: Инициализация `ts.Program`, кастомный `CompilerHost`, обход AST

### src/analyzer/resolver.ts
- **Зависимости**: `typescript`, `node:path`, `./config/types.ts`
- **Ответственность**: Resolution logic, маппинг `@scope/*`, обход баррелей

### src/graph/builder.ts
- **Зависимости**: `./config/types.ts`, `./analyzer/parser.ts`
- **Ответственность**: Сборка объекта `Graph` из распарсенных данных

### src/exporter/json.ts
- **Зависимости**: `node:fs`, `./config/types.ts`
- **Ответственность**: Сериализация графа, запись файла

### src/server/index.ts
- **Зависимости**: `bun`, `node:path`, `./frontend/*`
- **Ответственность**: HTTP сервер, раздача статики, автооткрытие браузера

### src/frontend/*
- **Зависимости**: `cytoscape` (через CDN)
- **Ответственность**: Визуализация, интерактивность

## 🔧 Ограничения TypeScript Compiler API

1. **Требуется tsconfig.json**: Без него невозможно создать `ts.Program`
2. **Статический анализ только**: Динамические импорты `import()` не отслеживаются
3. **Типы не влияют на граф**: Только синтаксические импорты/экспорты
4. **Path mappings**: Необходимо читать из `tsconfig.json` (`compilerOptions.paths`)
5. **SkipLibCheck**: Игнорируем `.d.ts` файлы для производительности

## 🚀 Производительность

- **Ожидание**: 100k файлов < 30 секунд
- **Оптимизации**:
  - Параллельный обход пакетов (не реализовано в v1)
  - Игнорирование `node_modules`, `dist`, `build`
  - Кастомный `CompilerHost` без лишних операций
  - Один `ts.Program` на пакет (не на весь монорепозиторий)

## 🔒 Обработка ошибок

- **Silent skip**: Ошибки парсинга отдельных файлов не прерывают процесс
- **Логирование**: Все ошибки выводятся в консоль (stderr)
- **Циклические зависимости**: Не обрабатываются, только логирование
