# ts-dep: CLI для статического анализа зависимостей в TypeScript-проектах

## 🎯 Проблема
В крупных NX монорепозиториях сложно отслеживать зависимости между пакетами. Разработчики сталкиваются с:
- Невидимыми циклическими зависимостями
- Неочевидными связями между модулями
- Сложностями при рефакторинге из-за отсутствия визуальной карты проекта
- Отсутствием инструмента для быстрого понимания структуры импортов

## 🔧 Решение
CLI-утилита `ts-dep`, которая:
1. Сканирует TypeScript-проект (особенно NX монорепозитории)
2. Строит граф зависимостей на уровне файлов и пакетов
3. Предоставляет интерактивную визуализацию через браузер
4. Экспортирует данные в машиночитаемом формате (JSON)

## 📦 Границы фичи (Scope)

### Включено:
- Парсинг статических `import`/`export` утверждений через TypeScript Compiler API
- Обнаружение пакетов в `packages/**/package.json`
- Resolution импортов вида `@scope/package` → конкретные файлы
- Обход баррель-файлов (`index.ts`) для поиска реальных источников
- Генерация графа с узлами (файлы/пакеты) и рёбрами (импорты)
- Интерактивная HTML-визуализация с Cytoscape.js
- Поиск и фильтрация узлов по имени и глубине
- Клик по узлу → переход к исходному коду (file:// protocol)
- Автооткрытие браузера при запуске сервера

### Исключено (на данном этапе):
- Обработка циклических зависимостей (только логирование)
- Динамические импорты (`import()`)
- Анализ зависимостей внутри `node_modules`
- Инкрементальное сканирование / кэширование
- Поддержка других языков (только TypeScript)
- Веб-интерфейс с фреймворками (только vanilla JS)

## 🏁 Критерии успеха
1. Команда `bun run src/cli/index.ts` успешно сканирует тестовый проект
2. Генерируется валидный `graph.json` с узлами и рёбрами
3. Сервер запускается на порту 8080, браузер открывается автоматически
4. Визуализация отображает граф с `cose` layout
5. Фильтры работают корректно (поиск по имени, фильтрация по глубине)
6. Клик по узлу открывает файл в редакторе (или показывает путь)

## 📋 Ограничения
- Максимум 100k файлов без кэша (ожидаемое время сканирования < 30 сек)
- Только статические импорты/экспорты
- Требуется наличие `tsconfig.json` в корне проекта
- Конфигурация через единственный файл `ts-dep.config.ts`
